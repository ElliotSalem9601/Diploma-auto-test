# Автоматизация тестирования функционала "Личные события" в расписании преподавателя

## О проекте

Данный проект содержит автоматизированные тесты (UI и API) для проверки функциональности **"Личные события"** на вкладке "Расписание" портала преподавателя Skyeng.

Проект выполнен в рамках **дипломной работы** по курсу "Автоматизация тестирования".

### Ссылка на финальный проект по ручному тестированию

[Курсовая работа: Тестирование функционала "Личные события"](https://elliotsalem9601.yonote.ru/share/c5147b1d-c7f2-4428-8bde-53912e961e8b)

---

## Цель автоматизации

Автоматизировать проверку CRUD-операций с личными событиями, их отображение и интеграцию с расписанием уроков, включая воспроизведение найденных в ходе ручного тестирования багов.

---

## Объем автоматизации

### UI тесты (5 шт.)
1. Добавление личного события (позитивный сценарий)
2. Валидация названия (ограничение 40 символов)
3. Редактирование названия события (проверка бага P2)
4. Удаление события (проверка бага P3)
5. Приоритет отображения (уроки выше личных событий)

### API тесты (5 шт.)
1. Создание события в будущем (позитивный тест)
2. Создание события в прошлом (потребность Анастасии Петровны)
3. Создание события с невалидной длительностью (негативный тест)
4. Редактирование события
5. Получение списка событий за конкретную дату

## Технологии

| Технология | Назначение |
|------------|------------|
| **Python 3.9+** | Язык программирования |
| **Selenium** | UI автоматизация |
| **Requests** | API автоматизация |
| **Pytest** | Фреймворк для тестирования |
| **Allure** | Генерация отчетов |
| **Page Object Model** | Паттерн для UI тестов |
| **python-dotenv** | Управление переменными окружения |
